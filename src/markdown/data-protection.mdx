## 2. Erhobene Daten

Wenn Sie ein Benutzerkonto anlegen, werden folgende personenbezogene Daten verarbeitet:

### 2.2 Hochgeladene Transaktionsdaten (CSV-Dateien)

Sie können CSV-Dateien mit Konto- und Depottransaktionen hochladen. Diese Daten werden **unmittelbar nach dem Upload anonymisiert**. Folgende Felder werden überschrieben:

#### Konto-Transaktionen

- „BIC / BLZ“ → `"anon"`  
- „IBAN / Kontonummer“ → `"anon"`  
- „Auftraggeberkonto“ → `"anon"`  
- „Konto“ → `"anon"`

#### Depot-Transaktionen

- „Depot“ → `"anon"`

Eine personenbezogene Zuordnung ist danach nicht mehr möglich.

## 3. Speicheroptionen für Transaktionsdaten

Sie haben die Wahl zwischen zwei Speicheroptionen:

### 3.1 Lokale Speicherung (Browser)

Wenn Sie die Option **Lokale Speicherung** wählen, werden Ihre anonymisierten Transaktionsdaten in der `IndexedDB` Ihres Browsers gespeichert (Pfad: `keyval-store/raw_transaction_data`). Diese Daten verbleiben ausschließlich auf Ihrem Gerät.

### 3.2 Server-Speicherung (geräteübergreifend)

Wenn Sie die Option **Server-Speicherung** wählen, werden Ihre anonymisierten Transaktionsdaten verschlüsselt auf unserem Server gespeichert. Der zur Entschlüsselung erforderliche Schlüssel wird lokal in Ihrer `IndexedDB` abgelegt (`keyval-store/encryptionKey`) und ist exportierbar. Ohne Zugriff auf Ihren Browser bleibt der Schlüssel Dritten unzugänglich.
